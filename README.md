# معلومات مشروع وتطبيق كيان النهضة العقارية - (Context Prompt for AI)

هذا الملف مصمم ليتم استخدامه كـ "برومبت" (Prompt) أو سياق (Context) لأي ذكاء اصطناعي سيقوم بمساعدتك في تطوير، تعديل، أو إضافة ميزات جديدة لهذا المشروع مستقبلاً. يعطيه هذا الملف فهماً كاملاً لهيكلية المشروع وتقنياته.

---

## 📌 معلومات عامة عن المشروع
- **اسم المشروع:** كيان النهضة العقارية (Kayan Al-Nahda)
- **مجال العمل:** شركة متخصصة وذات خبرة في التطوير العقاري.
- **اللغة الأساسية:** العربية (RTL).
- **الهدف من الموقع:** استعراض المشاريع العقارية (فلل، شقق، عمائر)، عرض المقالات/الأخبار، وتوفير وسائل تواصل للعملاء (مثل الواتساب).

## 🛠 حزمة التقنيات المستخدمة (Tech Stack)
- **الواجهة الخلفية (Backend):** Laravel (إطار عمل PHP).
- **الواجهة الأمامية (Frontend):** 
  - محرك القوالب: Laravel Blade
  - التنسيق: Tailwind CSS
  - التفاعلية في الواجهة: Alpine.js
  - المكونات الحية: Laravel Livewire (v3)

## 📁 الهيكلية والمجلدات الرئيسية
- **القالب الأساسي:** `resources/views/layouts/guest.blade.php`.
  - يتضمن خط المراعى (`Almarai`)، إعدادات الـ RTL، ميتا تاج (SEO و Open Graph)، وأكواد Google Tag Manager.
- **الصفحة الرئيسية:** `resources/views/welcome.blade.php`.
  - تحتوي على أقسام تفاعلية يتم استدعاؤها عبر Livewire أو كأجزاء (partials) مثل: `about-section`، `section-project`، و `section_blogs`.
- **مكونات Livewire:** `resources/views/livewire/`
  - مثل `section-project.blade.php` (لعرض المشاريع بالصفحة الرئيسية).
  - مثل `projects.blade.php` (للوحة تحكم الأدمن لإنشاء وإدارة المشاريع).
- **الصفحات الأخرى:** 
  - عرض كل المشاريع: `resources/views/projects.blade.php`.

## 🏗 النماذج والميزات (Models & Features)

### 1️⃣ إدارة النظام والمستخدمين (System & Users)
- **`App\Models\User`**: المستخدمون العاديون للنظام (يمتلكون: `name`, `email`, `password`).
- **`App\Models\Admin`**: الإداريون المتحكمون بلوحة التحكم (`name`, `email`, `password`).
- **`App\Models\AppSettings`**: نموذج مخصص لإعدادات النظام العامة للموقع.
- **`App\Models\Visitor`**: الزوار المهتمون الذين يرسلون استفسارات (نموذج التواصل السريع)، يحتوي على: (`first_name`, `last_name`, `phone`, `email`, `form_name`).

### 2️⃣ المشاريع العقارية والوحدات التابعة لها (Projects & Properties)
- **`App\Models\Project`**: المشروع الرئيسي الشامل (عمارة، فيلا، ...).
  - **أبرز الحقول:** الاسم (`name`)، الوصف (`description`)، الحالة (`status`)، النوع (`project_type`)، الموقع (`location`)، ومسار غلاف المشروع (`image_url`).
  - **حالات المشروع المدعومة:** 'جديد', 'تحت الانشاء', 'تم البيع', 'محجوز بالكامل'.
  - **منطق العرض في الواجهة:** يتم تغيير عرض البطاقة كلياً بناءً على الحالة (مثلاً تفعيل اللون الأبيض والأسود والنص العائم الأخضر وإلغاء إمكانية النقر إذا كانت حالة المشروع 'تم البيع').
  - **العلاقات:** يمتلك العديد من العقارات أو الوحدات (`Properties`) وصورة إضافية تابعة عبر (`ImageProject`).
- **`App\Models\ImageProject`**: صور داعمة للمشاريع وتشمل ترتيبها لتكوين معرض معماري (`url`, `order_by`, `project_id`).
- **`App\Models\Properties`**: الوحدات الفردية داخل أو التابعة للمشروع الأساسي (مثل الشقق داخل مبنى).
  - **أبرز الحقول:** تسعيرة وعروض الوحدة (`price`, `offer`)، تفاصيل الغرف (`rooms`, `bathrooms`, `living_rooms`, `mainds_room`, `driver_room`)، تفاصيل إضافية (`area`, `type`, `doors`, `parkings`, `facade`, `furniture`)، وروابط اليوتيوب لشرح الوحدة ومراحل البناء.
  - **العلاقات:** تنتمي لـ `Project`، وتملك معرض صور خاص بها عبر `ImageProperties`.
- **`App\Models\ImageProperties`**: صور عرض خاصة بكل وحدة/عقار داخل المشروع الرئيسي (`url`, `order_by`, `properties_id`).

### 3️⃣ الأخبار والمحتوى الديناميكي المنشأ (Blogs & Dynamic HTML)
- **`App\Models\Blog`**: المدونات والأخبار (يحتوي على: `title`, `image_post`).
  - **كيفية بناء المحتوى:** يعتمد المقال على هيكل شجري لتوليد أجزاء المقال ديناميكياً من خلال علاقات متعددة بـ `HtmlTag`.
- **`App\Models\HtmlTag`**: أداة بناء محتوى المقالات باستخدام هيكل شجري لعناصر HTML لإنشاء المقالات بشكل مرن (Page Builder).
  - **الحقول الأساسية:** نوع الوسم (`tag_name`)، تنسيقات Tailwind (`classes`)، الترتيب المعماري للوسوم (`order_by`)، الخصائص الإضافية (`tag_attributes`). يرتبط بالمقال الأم (`blog_id`) وعلامة HTML أصلية أحياناً (`html_tag_id`).
- **`App\Models\HtmlTagText`**: المحتوى النصي المكتوب (`content`) بداخل كل وسم ديناميكي.

## 🎨 هوية التصميم (Design System)
- الألوان الرئيسية للمشروع ترتكز على درجات اللون الأخضر (مثل: `#498E49` و `#49A035`).
- خلفيات أقسام الموقع تستخدم الألوان الهادئة مثل `#EAF5EA` أو `#f8f9fa`.
- البطاقات تعتمد على تصميم حديث يشمل انحناءات ملساء (`rounded-[2rem]`)، تظليل (`shadow-lg / shadow-2xl`)، وتأثيرات التحرك والانتقال السلس (`transition-all duration-500`).

## ✍️ تعليمات للذكاء الاصطناعي (AI Instructions) عند التحديث أو التطوير المستقبلي:
1. **الالتزام بالتصميم:** عند إنشاء أي عنصر جديد، يجب أن يطابق تصميم العناصر الحالية (استخدم نفس الألوان والانحناءات Tailwind classes).
2. **استخدام Livewire و Alpine:** عند الحاجة لإضافة تفاعلات ديناميكية بدون إعادة تحميل الصفحة، اعتمد على Alpine.js للواجهة (UI interactions) وعلى Livewire للتواصل مع الخادم.
3. **الدعم العربي والجوال:** أي تحديث يجب أن يدعم الكتابة من اليمين لليسار (`RTL`) ويكون متجاوباً تماماً مع جميع الأجهزة (Mobile First).
4. **تجنب كتابة أكواد CSS أو JS منفصلة:** حاول قدر الإمكان استخدام Tailwind classes لتنسيق العناصر، واستخدم Alpine.js للأكواد البرمجية التفاعلية القصيرة مباشرة على عنصر الـ HTML، أو اكتب أكواد Livewire كـ PHP Classes بدلاً من jQuery وأخواتها.
